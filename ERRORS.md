# Errors & Issues Log

> Track all errors, bugs, and issues encountered during development.
> Format: E-NNN | Layer | Status | Description | Fix

---

## How to use this file

When you hit an error:
1. Add a new entry with the next number (E-001, E-002, ...)
2. Fill in Layer, Status (OPEN / RESOLVED / WONTFIX)
3. Paste the error message
4. Document the fix once resolved

This file becomes your debugging history and is useful for interviews — shows you can systematically debug production issues.
